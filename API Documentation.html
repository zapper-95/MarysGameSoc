Mary's Game Soc

The following GET and POST methods are for two entities: requests and reviews. Below you can see an example of each one of these requests with explanation as to what they do and what the search parameters correspond to.

GET

Get Game Requests

http://127.0.0.1:8090/game/request?game_request=o

game/request is a GET method designed to return all the names of all the games requested according to a certain characteristic, namely whether the gameID of the requested games contains the search parameter of the request.

In the example below, the search parameter (game_request) is set equal to o. This means when the request gets to the server it will extract the search parameter o and then iterate through each game request. For each game request where its gameID contains an o, the gameID will be added to a list. This list is what will then be sent back to the client.

For example the following would be sent back for the request above:

[ "Avalon", "Saboteur Gold", "Terraforming Mars", "Dobble", "King Domino", "Cluedo", "Colt Express", "Port Royal"]

Notice how each of these gameID's contains an "o" in them

PARAMS
game_request
o


Example Request
Get Game Requests
curl --location --request GET 'http://127.0.0.1:8090/game/request?game_request=o'
GET

Get game request details

http://127.0.0.1:8090/game/details?game=Avalon

game/details is a GET method designed to return specific details for a particular game request. The search parameter for this request indicates the specific game request that further details are wanted for. Once this particular game request is located, these specific details will be returned to the client as JSON data. These specific details include the gameID, who submitted the game, the date of submission and whether the person who the submitted the game request has a copy of the game.

For the example above, the following JSON data would be returned to the client

{
    "gameID": "Avalon",
    "submittedby": "James",
    "date": "19/1/2022",
    "HasaCopy": true
}

PARAMS
game
Avalon


Example Request
Get game request details
curl --location --request GET 'http://127.0.0.1:8090/game/details?game=Avalon'
POST

Add a game request

127.0.0.1:8090/game/new

game/new is a POST method designed to add a new game request to the JSON file containing all the game requests.

To make a post request the following attributes will be needed for the body:

gameID: What exactly is the name of the new game you are wanting to request?

submittedby: Who is submitting this new game request?

HasaCopy: Does the person submitting the game request have a copy of the game preferably so they can bring it along to the society meetings.

After the client has sent this post request, the server will send one of two types of
responses depending on the body parameters:

Request Successfully Received:
In which case all has gone well and the body parameters have been stored
successfully.

Game Already Requested:
In the instance where the client requests the same game as one that has already
been requested. In this case, the clients post request will have had no effect
on the data stored on the server.

HEADERS
Content-Type
application/json
BODYraw
{"gameID": "Azul", "submittedby": "Jon" , "HasaCopy": false}


Example Request
Add a game request
curl --location --request POST '127.0.0.1:8090/game/new' \
--header 'Content-Type: application/json' \
--data-raw '{"gameID": "Azul", "submittedby": "Jon" , "HasaCopy": false}'
GET

Get Game Reviews

http://127.0.0.1:8090/game/reviews?game_review=d

game/reviews is a GET method designed to return all the reviews of the games requested given the search parameter is contained within the name of the requested game.

In the example below, the search parameter (game_review) is set equal to d. This means when the request gets to the server it will extract the search parameter d and then iterate through each game review. For each review where its gameID contains a d, {gameID + " by " + submittedby} will be added to a list. This list is what will then be sent back to the client.

For example the following would be sent back for the request above:

[
    "Dixit by Amy",
    "Dixit by Michael",
    "Cluedo by Sally",
    "Dobble by Joseph",
    "Cluedo by Harry",
    "King Domino by Alex"
]

Notice how each of the names of the games (or gameIDs) contains an "d" in them

PARAMS
game_review
d


Example Request
Get Game Reviews
curl --location --request GET 'http://127.0.0.1:8090/game/reviews?game_review=d'
GET

Get review details

http://127.0.0.1:8090/review/details?review=Avalon by James

review/details is a GET method designed to return specific details for a particular game review.

The search parameter for this request indicates the specific game review being requested so that further details are given. It includes both the name of the game which was reviewed and the person who reviewed it.

Once a particular game review is located, the details of the review will be returned to the client as JSON data. More precisely this is the name of the game being reviewed or the gameID, who submitted the review, what they rated the game (from 1 to 10) and what the comment on the game is.

For the example above, the following JSON data would be returned to the client

{ "gameID": "Avalon", "submittedby": "James", "rating": 9, "comment": "Great fun and I managed to win!"}

PARAMS
review
Avalon by James


Example Request
Get review details
curl --location --request GET 'http://127.0.0.1:8090/review/details?review=Avalon%20by%20James'
POST

Add a game review

http://127.0.0.1:8090/review/new

review/new is a POST method designed to add a new game review to the JSON file containing all the game reviews.

To make a post request the following attributes will be needed for the body:

gameID: What exactly is the name of the new game you are wanting to review?

submittedby: Who is submitting this new game review?

rating: How good is this game, from 1 to 10?

comment: What exactly did you think about the game?

After the client has sent this post request, the server will send one of three types of
responses depending on the body parameters:

Review Successfully Received!
In which case all has gone well and the body parameters have been stored
successfully.

Rating is not a number in the range 1 to 10:
In the instance where the client posts a number rating which is not in the range 1 to 10 inclusive.

Unfortunately that game has not been requested yet to play so it cannot have a review: In the instance where the game that has been reviewed has not been requested for the society to play yet.

HEADERS
Content-Type
application/json
BODYraw
{"gameID": "Cluedo", "submittedby": "Sally", "rating": 1, "comment": "turned out to be professor plum!"}


Example Request
Add a game review
curl --location --request POST 'http://127.0.0.1:8090/review/new' \
--header 'Content-Type: application/json' \
--data-raw '{"gameID": "Cluedo", "submittedby": "Sally", "rating": 1, "comment": "turned out to be professor plum!"}'
